# Script ScreenStudio - RepoScope

Durata target: 40-50 secondi

## Preparazione

- Apri RepoScope su `/repositories`.
- Usa una finestra browser pulita, senza pannelli laterali visibili.
- Tieni gia caricati i dati GitHub.
- Parti dalla vista repository, con la sidebar filtri visibile.
- Scegli un repository con dati ricchi, ad esempio uno con Actions, release, fork e traffico.

## Sequenza Video

### 0-5s - Apertura sulla dashboard

**Azione:** mostra la vista `/repositories`, senza cliccare subito.

**Voce:**
> Questo è RepoScope, una dashboard locale per osservare e gestire i tuoi progetti GitHub da un unico posto.

### 5-12s - Filtri e vista repository

**Azione:** muovi leggermente il mouse sulla sidebar filtri e poi sulle card repository. Non aprire ancora modali.

**Voce:**
> Qui posso filtrare repository, issue e pull request per organizzazione, linguaggio, stato e visibilità, mantenendo una vista compatta e ordinabile.

### 12-18s - Apertura dettaglio repository

**Azione:** clicca su una card repository.

**Voce:**
> Aprendo un repository entro in una scheda completa, con URL stabile e sezioni navigabili.

### 18-28s - Overview e grafici

**Azione:** resta su `Overview`, mostra topic, metriche e grafici. Se serve, scorri poco verso i grafici.

**Voce:**
> Nella panoramica vedo metadati, contributor, linguaggi e grafici di trend per stelle e fork, utili per capire subito l'andamento del progetto.

### 28-36s - Actions, PR e Issues

**Azione:** clicca `Actions`, poi `PRs`, poi `Issues`. Fermati mezzo secondo su ogni tab.

**Voce:**
> Le tab operative mostrano GitHub Actions, pull request e issue. Ogni lista carica dieci elementi alla volta e si sfoglia con paginazione.

### 36-44s - Releases, Forks e Traffic

**Azione:** clicca `Releases`, poi `Forks`, poi `Traffic`. Su `Forks`, mostra rapidamente i controlli di ordinamento.

**Voce:**
> Posso consultare release, fork ordinabili e traffico senza uscire dal flusso, con link diretti alle pagine GitHub quando serve approfondire.

### 44-50s - Chiusura

**Azione:** chiudi la modale e torna alla griglia repository.

**Voce:**
> RepoScope trasforma GitHub da tante pagine separate in un cockpit operativo per capire cosa sta succedendo e dove intervenire.

## Note Di Ripresa

- Mantieni zoom browser al 100%.
- Evita movimenti rapidi del mouse.
- Non aprire link esterni durante il video.
- Se una tab ha pochi dati, usa un repository diverso prima di registrare.
- Il ritmo ideale è: un clic ogni 4-6 secondi, senza scroll lunghi.
